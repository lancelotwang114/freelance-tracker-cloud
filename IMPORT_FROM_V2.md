# 從 v2（單機版）搬資料到 v3（雲端版）— 完整步驟

> ⏱️ 預估時間：**3-5 分鐘**（v3.22.1 起改自動匯入）
> 🛡️ 安全性：**v2 不會被動到**，v3 是獨立的瀏覽器資料 + 獨立的 Drive 檔案。可以隨時回頭。
> 📦 適用：v2.10.x 任何版本
> 🆕 v3.22.1：匯入時**自動帶入收款帳號 + 通知偏好 + schema 自動升級 + 存摺照片自動遷移**

---

## 開始前的核對清單

- [ ] 我有 v2 的網址 <https://lancelotwang114.github.io/freelance-tracker/> 可以打開
- [ ] 我有一個 Google 帳號（任何 gmail 都可以）
- [ ] 我有 5-10 分鐘不被打擾的時間
- [ ] 建議：**先把 v2 匯出兩份備份**（一份留底、一份匯入用）

## Phase 1：在 v2 匯出 JSON 備份（2 分鐘）

1. 開 **v2**：<https://lancelotwang114.github.io/freelance-tracker/>
2. 切到 **設定** 分頁
3. 找到「💾 資料備份」card → 展開
4. 按 「📤 匯出 JSON 備份」
5. 瀏覽器會下載一個檔案，名字類似 `freelance-tracker-backup-2026-05-01.json`
6. **把檔案存到桌面**（或任何容易找的地方）
7. **建議**：再按一次匯出，把第二份 rename 成 `freelance-tracker-LEGACY.json` 收進雲端硬碟備份夾裡

> ⚠️ **不要按到「跨裝置設定檔」匯出**，那是另一個東西（含 token / API URL），v3 不需要也不能用。

---

## Phase 2：開 v3 + Google 登入（2 分鐘）

1. 開新分頁 → <https://lancelotwang114.github.io/freelance-tracker-cloud/>（**v3 雲端版**）
2. 等頁面載完，會看到歡迎引導
3. **點右上角「使用 Google 登入」按鈕**
4. 跳出 Google 授權對話框：
   - 選擇要用的 Google 帳號
   - 授權清單會包含「應用程式資料夾」+ Google Calendar（先全部允許）
5. 登入完成，右上會顯示你的頭像 + email
6. 接著會跳一個 modal：「📅 要不要也同步到 Google 行事曆？」
   - **先按「先不要」**（之後可以再開，先把資料搬好）

> 🔍 注意：v2 跟 v3 部署在**同一個網域**（lancelotwang114.github.io），但 v3 的 localStorage 加了 `cloud-` 前綴隔離，**v2 資料不會被影響**。

---

## Phase 3：匯入 JSON（2 分鐘）— 🚀 v3.22.1 起一次到位

1. v3 切到 **設定** 分頁
2. 找到「💾 資料備份」card → 展開
3. 按 「📥 匯入備份」
4. 選 Phase 1 下載的 JSON 檔
5. 跳出確認對話框（內容會列出**所有將被自動處理的東西**）：
   ```
   準備匯入（v3.22.1：自動升級 + 偏好設定也一起搬）：

   • 業主 N 位（現有 0 位）
   • 案件 N 筆（現有 0 筆）
   • 收款帳號 N 個
   • 通知與提醒、收益目標等所有偏好設定

   📋 來源：schema v7（將自動升級到 v13）
   🖼️ 存摺照片將自動上傳到 Drive App Folder

   ⚠️ 匯入會覆蓋現有資料。確定？
   ```
6. 確認 → 看到「✓ 已匯入：業主 N 位 · 案件 N 筆 · 收款帳號 N 個 · N 項偏好」toast

### v3.22.1 自動處理的內容
- ✅ **業主**：含 contact / tags / 補新欄位
- ✅ **案件**：含 quantity / tags / payments[]（v6 之前舊版自動轉換）
- ✅ **收款帳號**：v2 簡單版自動 mapping 到 v3 完整版（個人資訊、發票資訊從 top-level userInfo 補）
- ✅ **selected paymentAccountId**：保留之前選定的帳號
- ✅ **通知與提醒**：所有 toggle + 天數參數
- ✅ **收益目標**：v3 → v3 之間 import 帶
- ✅ **schema migration**：自動跑 v6→v13（6 個版本一次升完）
- ✅ **存摺照片**：base64 自動上傳到 Drive 個別檔（背景跑、無需等待）
- ✅ **舊欄位自動補**：v2 沒有的 contact / tags / quantity 等都補預設值

## Phase 5：手動補的東西（剩兩項，2 分鐘）

v3.22.1 起大部分自動了，**只剩這 2 項真的需要手動**：

### 5-1. Google 行事曆同步（**無法自動**）

v2 是 Apps Script 中介，v3 直接打 Calendar API。兩套機制完全不同，無法 mapping。

1. 設定頁 → 「📅 Google 行事曆同步」 card 展開
2. 開啟 master toggle「啟用 Google 行事曆同步」
3. 從下拉選擇要同步的日曆（建議獨立建一個「外包」日曆）
4. 早報時段、要同步哪些事件依需求設定

### 5-2. 業主標籤 + 通訊錄（v3 新功能，可選）

v2 沒這些欄位，無法從 JSON 帶。但很值得補：

1. 切到 **業主** 分頁
2. 點任一業主進 detail 頁
3. 「🏷️ 標籤」加：VIP / 長期 / 拖款戶 等
4. 「📇 通訊錄」填：聯絡人 / 電話 / Email / 地址
5. v3 業主 detail 頁的智慧分析會利用這些資訊

> 💡 如果你業主多，先補主力業主的就好，其他遇到再補。

## Phase 7：保留 v2 作為備援（重要）

**不要立刻刪 v2 的資料！** 建議的觀察期：

| 觀察期 | 動作 |
|--------|------|
| 第 1 週 | 兩版並用：v3 為主、v2 偶爾開來確認 |
| 第 2 週 | 完全用 v3，但 v2 不動 |
| 第 3 週後 | 確認 v3 用得順 → 可考慮清 v2，或讓 v2 留著當備援也行 |

**永遠別做的事**：
- ❌ 看到 v3 順 → 立刻清 v2（萬一有 bug 沒地方退）
- ❌ 把 Phase 1 匯出的 JSON 刪掉（這是你的 ground truth 備份）

## 常見問題

### Q1: v3 匯入完成後，案件數對但金額有點對不上

**可能原因**：
- v2 是 schema v6，v3 已到 v13，v6→v7 的 migration 會把舊 `paid:true` 自動轉成 `payments[]` 陣列
- v3.6.1 的 demo bug 修就是同個原理（沒 payments 的 paid 案件會被算成 0 已收）
- 刷新頁面（Phase 4）讓 migration 跑後，金額會回正常

**驗證方法**：開瀏覽器 console（F12），看有沒有 `✓ Schema migrated` 訊息。

### Q2: 存摺照片匯入後變「⏳ 載入中」一直轉

**這是正常的，請等待 30-60 秒**。v3 會自動把 base64 存摺照片遷移到 Drive App Folder 個別檔（節省 tracker.json 大小）。遷移完成自動換成圖片。

如果一直轉不過來：
1. 確認已 Google 登入（top bar 顯示頭像）
2. 重整頁面再等 30 秒
3. 檢查 console 有沒有 error

### Q3: 通知與提醒設定都重置了

**對，這是預期行為**。v3 的提醒 schema 不一樣（每個提醒有兩個通道），手動重設一次即可。

### Q4: v3 沒辦法用，能不能回 v2？

可以。**完全沒被動到**：
1. 直接開 <https://lancelotwang114.github.io/freelance-tracker/>（v2 URL）
2. v2 的 localStorage 跟 v3 是隔離的（v3 用 `cloud-` 前綴）
3. 一切如常

如果要把 v3 的最新資料帶回 v2：
1. v3 → 匯出 JSON
2. v2 → 匯入 JSON（v2 會吃 schema v7，更高版本可能會有些欄位無視）
3. 部分新功能（tags / contact / invoiceHistory）在 v2 不顯示，但資料還在 JSON 裡，將來回 v3 還能用

### Q5: 兩版能同時用嗎？

可以，**但不建議長期**。資料分別跑兩條，會分歧。建議選一邊主用。
