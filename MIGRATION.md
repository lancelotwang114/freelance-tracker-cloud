# 給原使用者：v3 雲端版有什麼不一樣？

> 你正在用的 v2 還是穩定版、不會停。但 v3 走了完全不一樣的路，**設定變超簡單，功能多很多**。
> 想試試？→ <https://lancelotwang114.github.io/freelance-tracker-cloud/>

---

## 5 秒看懂差異

| | v2（單機版） | v3（雲端版） |
|---|---|---|
| **後端** | 你要自己架 Apps Script + Sheet | 不用，純前端打 Google Drive |
| **登入** | 貼 API URL + 貼 token | Google 一鍵登入 |
| **設定時間** | 30 分鐘起跳 | 10 秒 |
| **跨裝置同步** | 兩台都要貼一樣的 token | 同 Google 帳號自動同步 |
| **衝突處理** | ABC 三選一蓋掉 | 三方合併（盡量不蓋資料）+ 真衝突逐筆挑 |
| **行事曆同步** | 要弄 Apps Script | 直接接 Google 行事曆 API |

## v3 新功能（v2 沒有的）

### 🎯 業主管理升級成 CRM-lite
點業主進**詳細頁**，看到：
- 累計收入、案件數、平均收款週期、待收餘額
- 過去 12 個月貢獻趨勢圖
- 通訊錄（聯絡人 / 電話 / Email / 地址）
- 案件歷史時間軸
- **智慧分析提示**：90 天沒下單 / 拖款比平均慢 5 天 / 年度貢獻佔 50% 集中度警告

### ⏱️ 全局計時器
從案件 modal 內計時 → top bar 常駐 widget。
- 切到任何分頁 / 開任何 modal / 瀏覽器關了再開都不中斷
- 結束計時自動把累積小時加到該案件工時欄

### 🎯 達成率 + 智慧分析
- 設月目標 / 年目標 → 進度條視覺化
- 線性預測「依目前進度，本月將達 NT$X / 距離達標差 NT$Y」
- 業主集中度警告、churn 警告、拖款指數

### 📦 請款單歷史
每次匯出 PDF / 圖 / 列印都自動留紀錄，含 status：
- ⚪ 待寄出 → ✉️ 已寄出 → 💰 部分收款 → ✅ 已收齊
- 一鍵重發（同樣案件再產一張）

### 🗂️ 案件 5 種視圖
從原本的單一列表升級成可切換：
- **📋 完整**（同 v2 列表）
- **📑 緊湊**（一行一筆，密度提升 3 倍）
- **📊 報表**（spreadsheet 表格、預設）
- **🎴 卡片**（grid 排列）
- **🗂️ 看板**（4 column 拖曳改狀態）

### 🏷️ 標籤系統升級
業主 + 案件都可以打多標籤（VIP / 長期 / 拖款戶 / 設計 / 動畫 …）
- chip 介面 + Enter 新增
- 共用標籤池、自動建議

### ↶ Undo（多步撤銷）
- 刪錯案件 / 業主 / 批次操作 → **Ctrl+Z 隨時復原**
- 最多 30 步、Ctrl+Shift+Z 重做

### 📅 Google 行事曆雙向同步
案件 + 5 種提醒 + 每日早報自動推到 Google 日曆，**手機收原生通知**。
- 9 種提醒類型，每個自選通道（桌面通知 / 行事曆 / 兩者）

### 📱 手機優化
- 案件 row 左滑 → 標完成、右滑 → 標收款
- 行事曆拖卡片 → 改日期
- 響應式 5 種視圖

### 其他

## 怎麼開始體驗

1. 開 <https://lancelotwang114.github.io/freelance-tracker-cloud/>
2. 點右上「Google 登入」（同個 Google 帳號就好）
3. 點「載入範例」看 6 業主 / 35 案件 / 3 收款帳號的完整展示
4. 玩夠了清空、開始用

**v2 跟 v3 完全獨立，本機資料不會打架**（v3 加了 `cloud-` 前綴隔離）。
試 v3 的時候 v2 不會被動到、隨時可以回頭。

## 我該轉 v3 嗎？

**建議轉 v3**，如果你符合：
- 想擺脫每次換裝置都要貼 token / API URL
- 想要更多功能（CRM、計時器、達成率、看板等）
- 已經有 Google 帳號（基本上都有）
- 不在意定期登入（Google token 1 小時過期，再點一次就好）

**繼續用 v2**，如果你符合：
- 已經把 v2 跑得很順、不想改變工作流
- 不想用 Google 帳號登入
- 想完全自己掌控後端

## 資料怎麼搬？（從 v2 → v3）

目前**沒有自動匯入工具**（v2 是 Apps Script + Sheet，v3 是 Drive JSON，格式不同）。

如果想搬：
1. v2 設定頁 → 「💾 資料備份」 → 匯出 JSON 備份
2. v3 設定頁 → 「💾 資料備份」 → 匯入備份
3. 部分欄位（schema v6 → v13）會自動 migration

實際測試前**先在 v2 匯出一份備份留底**，避免出意外。
